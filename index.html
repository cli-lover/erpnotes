Odoo ERP

Odoo is a robust, open-source Enterprise Resource Planning (ERP) software, designed with a modular approach that allows it to be customized to meet the diverse needs of businesses across various industries.

In this blog, I will share my insights, experiences, and the knowledge I've gained while exploring Odoo, with the aim of leveraging its capabilities for practical business solutions.

Understanding Odoo's Architecture

Odoo is built on a multi-tier architecture, consisting of the following three key layers:

  • Application Tier: This layer handles the core logic and processing. Odoo's core and its various modules are developed using Python, and additional modules can be installed based on user needs. It relies on an ORM (Object-Relational Mapping) that interfaces with the database.
  • Presentation Tier: The user interface layer, which is accessed via a web browser. It serves dynamic content to users using web technologies such as HTML, JavaScript, and XML, allowing users to interact with Odoo’s various applications.
  • Database Tier: Odoo's database layer is powered by PostgreSQL. The ORM maps Python objects to database tables, and PostgreSQL ensures that the data is securely stored and managed.

Odoo overview

Odoo is a web-based platform that runs through your browser. It has two main parts: the Client side (Web Client) and the Server side (Application and Database).

Client Side (Web Client)

The Client side is where the magic happens for users, using HTML, CSS, and JavaScript to create the presentation layer. In Odoo, JavaScript and SCSS help make the pages look great. For an even more structured and modular design, Odoo uses its own front-end framework called "OWL" (Odoo Web Library). It makes building UI a lot more organized and efficient.

Server Side

The Server side is where all the heavy lifting happens. It has the Logic Layer, which is built with Python, and the Data Layer, which uses PostgreSQL for storing and managing data.

How Do They Talk to Each Other?

The Presentation Layer and the Logic Layer communicate via RPC (Remote Procedure Call). This ensures everything on the interface reacts smoothly based on the logic running in the background. The Logic Layer and Database Layer work together using Odoo’s ORM (Object Relational Mapping), which translates Python code into SQL commands to keep things secure and modular.

The Object-Relational Mapping (ORM) in Odoo

Odoo’s ORM is a key part of its architecture. It abstracts the database layer, allowing developers to interact with Python objects rather than writing SQL queries. When a model is defined in Odoo, the ORM automatically generates the corresponding database table. This makes it easier to enforce constraints and relationships while maintaining the flexibility of Python.

Why is Odoo "Modular"?

Being modular means that Odoo is built up of individual components called modules. Everything in Odoo is a module, and these modules can work independently or together.

What are Odoo Modules?

Modules are the building blocks of Odoo. Here are a few examples:

  • Application Modules: Complete business applications that run processes like accounting, inventory, or CRM.
  • Technical Modules: These are the behind-the-scenes tools that help with integrations, web services, and other technical tasks.
  • Theme Modules: Front-end modules that provide themes, layouts, and styling for the Odoo interface.
